Use Cases

User Onboarding & Reward Redemption​

  1. User joins SuperJ and gets a JM DID.
  2. Engages in surveys or enterprise tasks.
  3. Earns JM Coin (wrapped in a fungible token).
  4. Redeems for gift cards or other assets.

Enterprise & Identity Verification​

  1. Businesses onboard users via JM DID.
  2. DID credentials are stored on Layer 2.
  3. Users can share credentials across multiple platforms.
